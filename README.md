# TimeKepper (タイムキーパー)

## 概要
イベント運営における、進行スケジュールの遅延状況をリアルタイムに可視化・共有するための Slack Bot です。

## 開発の背景
私が所属している文化祭実行委員会で、イベント当日の進行管理において、「今、予定より何分遅れているか（何分巻いているか）」を正確に把握し、スタッフ全員に即座に共有する必要がありました。
従来は手計算や口頭での共有に頼っていましたが、計算ミスの防止と共有のスピードアップを目的として、スプレッドシートと連携したこのツールを開発しました。

## 主な機能
- **進行状況の自動計算**: `/tk [部屋番号]` コマンドを打つだけで、スプレッドシートの予定時刻と現在時刻を照合し、「○分押し」「○分巻き」「定刻通り」を即座に算出します。
- **柔軟な表示**: 部屋番号やイベント内容に応じて、「開始時刻」と「終了時刻」のラベルを自動で切り替えます。
- **ログの無効化（取り消し）機能**: 誤送信時にスレッドで「取り消し」と打つことで、元の投稿を打ち消し線で更新し、実行ログを残す仕組みを備えています。
- **表示の工夫**: ゼロ幅スペース（`\u200B`）を利用し、Slackの自動リンク化や予期せぬ整形を防ぎ、視認性を高めています。

## セットアップ
1. **Google スプレッドシートの準備**
   - 「index」という名前のシートを作成します。
   - A列：ID（部屋番号など）、B列：イベント名、C列：詳細、D列：予定時刻（HH:mm形式）を配置します。
2. **スクリプトプロパティの設定**
   - GASのエディタの「設定」から以下のプロパティを追加してください。
     - `SLACK_TOKEN`: Slack APIの Bot User OAuth Token
     - `SPREADSHEET_ID`: 連携するスプレッドシートのID
3. **デプロイ**
   - Webアプリとしてデプロイし、実行ユーザーを「自分」、アクセス権限を「全員（匿名）」に設定します。
   - 発行されたURLをSlack Appの「Event Subscriptions」および「Slash Commands」に設定します。

## 使い方
1. Slackのチャンネル上で `/tk 101` （101はスプレッドシートに登録したID）を実行。
2. Botが現在の遅延状況を計算し、チャンネル内に投稿します。
3. 投稿を消したい場合は、その投稿のスレッドに「取り消し」と入力します。

## 技術スタック

- Slack API (Events API / Slash Commands)
- Google Sheets API
